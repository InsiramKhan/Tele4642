Machine Learning Integration

Why Use Machine Learning?

Traditional firewalls rely on static rule sets. In smart-home environments with dynamic and diverse traffic, this approach fails to detect novel or evolving threats. By using supervised machine learning, IntelliHome SDN can classify network flows in real time and adapt to emerging patterns of malicious behavior.

Classifier Used

We use a RandomForestClassifier from the scikit-learn library. This ensemble-based model is well-suited for tabular flow data, resistant to overfitting, and fast enough for real-time inference in a network controller environment.

Feature Set

Each flow is described using the following extracted features:

  • Time Stamp
  • Source and Destination Ports
  • Protocol (e.g., TCP, UDP)
  • Machine Learning Decision
  • MUD Decision
  • Packet Length

Classifier Output

The classifier outputs a binary label for each flow:

  • 0 – Benign: normal communication
  • 1 – Malicious: anomalous or unauthorized traffic
This decision is fed back to the SDN controller to either install, quarantine, or drop the flow.

Integration with Ryu Controller

After initial flow stats are gathered, the controller queries the trained ML model in-memory. If a flow is flagged as malicious, the controller immediately logs the event, removes any matching rule, and optionally broadcasts an alert via the REST API.
